# CLAUDE.md

## Project overview

Cork|Sec website — a static site for a monthly security meetup in Cork, Ireland (DEFCON Group DC021353, running since June 2013). Hosted on GitHub Pages at `https://corksec.github.io`.

No build step. All HTML, CSS, and JS are served as-is from the repo root.

## Site structure

- `index.html` — Home page. Welcome text + upcoming events loaded from `data/upcoming.json` via client-side JS.
- `past-talks.html` — Past events archive. Fetches `data/past-events.json` and talk slides/PDFs from the `CorkSec/website-docstore` GitHub repo via the GitHub API.
- `submit.html` — Talk submission info page.
- `src/styles.css` — All styles. Matrix-inspired black/neon-green theme using CSS custom properties (`--neon`, `--bg`, `--panel`, etc.). Monospace fonts throughout.
- `src/getting-into-info-sec.html` — Redirect page to external resources.

## Data files (`data/`)

All JSON. Same schema for upcoming and past events:

```json
{
  "title": "Cork|Sec 154",
  "date": "2026-04-14",
  "time": "19:00",
  "description": "...",
  "venue": "The Pavilion, Careys Lane, Cork City",
  "talks": [
    { "title": "TALK 1: Title - Speaker", "description": "Abstract text" }
  ],
  "meetupUrl": "https://www.meetup.com/corksec/events/..."
}
```

- `upcoming.json` — Auto-generated by the scraper. Upcoming events only.
- `past-events.json` — Archived past events (150+). Populated by the archive script.

## Scripts (`scripts/`)

### `fetch-events.py`

Scrapes upcoming events from `https://www.meetup.com/corksec/events/`. Extracts the `__APOLLO_STATE__` object embedded in Meetup's `__NEXT_DATA__` script tag. Parses event metadata (title, date, time, venue, description) and extracts individual talks from the description by matching `TALK N:` lines (which may be wrapped in markdown bold `**`). Writes to `data/upcoming.json`.

### `archive-events.py`

Moves events with past dates from `upcoming.json` into `past-events.json`. Deduplicates by date. Runs before the fetch script in the workflow.

## GitHub Actions (`.github/workflows/update-events.yml`)

Runs on the 1st and 15th of each month at 10:00 UTC (also manually via `workflow_dispatch`):

1. Archive past events
2. Fetch fresh upcoming events from Meetup
3. Commit and push changes to `data/` if any files changed

Uses Python 3.12. Has `contents: write` permission.

## How event data flows

```
Meetup.com event page
  -> fetch-events.py scrapes Apollo state
  -> parses events + talks from descriptions
  -> writes data/upcoming.json
  -> index.html JS renders event cards

When an event date passes:
  -> archive-events.py moves it to data/past-events.json
  -> past-talks.html renders it in the archive
```

## Key conventions

- Vanilla JS only (no frameworks, no build tools).
- All client-side rendering uses string concatenation in `<script>` blocks — no templating library.
- Talk slides/documents are stored in a separate repo (`CorkSec/website-docstore`), not in this repo.
- The scraper uses no external Python dependencies — stdlib only (`urllib`, `json`, `re`).
- CSS uses `var(--neon)` / `var(--bg)` / `var(--panel)` etc. for theming. Keep all colours in `:root`.
